Document Object Model (DOM)

Q1: What is DOM and what is the purpose of DOM?

DOM means Document Object Model which is a programming interface used for changing document structure, style and content dinamically. DOM represents the document as nodes and objects. To create dynamic and interactive web pages,developer use DOM.

Some purpose of DOM is given bellow

  • React to HTML events
  • Add HTML elements​
  • Remove HTML elements
  • Change HTML elements​
  • Add attributes​
  • Remove HTML attributes​
  • Change HTML attributes​
  • Change CSS stye properties

Q2: How will you select HTML elements using DOM? Name the DOM methods.

We can select elements using dom in five methods.

The methods are given bellow-

  • getElementById(): Select element which has the specified id
  • getElementsByClassName(): Select elements which has same class name
  • querySelector(): Select the first element in which condition matched
  • querySelectorAll(): Select all elements which match with condition
  • getElementsByTagName(): Select elements which has specified class name

Q3: What is event bubble?

Event bubble is the process of DOM manipulation by which an event in the DOM is first captured by the deepest child element which is the target of the event. After that the bubbles up through its parent elements until it reaches the topest element.

There are two working procedure of the event bubble. They are capturing and bubbling. For capture phase, the event travels down the DOM tree from top parent object to the target element. During the bubbling phase, the event travels back up the tree from the target element to the parent object.

Q4: What is a callback function and why will you use it?

Callback function is a type of function that is passed as an argument to another function, and is executed by that function when a certain event occurs or after another function has completed its execution.

The main reason for using callback functions is to allow for asynchronous or non-blocking behavior in JavaScript code. By passing a callback function as an argument, the calling function can continue executing other code while the callback function is being executed in the background.
